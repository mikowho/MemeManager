# MemeManager (表情包管理器) - 毕业设计 / 学习项目

一款基于 Android Jetpack Compose 开发的本地表情包管理工具。
本项目是 **niko** 与 **Gemini AI** 共同学习和探索 Android 开发的成果。
**严禁任何商业用途！** 仅供个人学习、交流和非营利性质的分享。

## ✨ 核心功能

*   **URL 直链下载**：输入 ZIP 链接，自动下载并解压。
*   **智能重命名**：支持中文 URL 解码，自动处理重名文件（提供覆盖或新建副本选项）。
*   **本地导入**：支持从手机文件系统导入 ZIP 表情包。
*   **分类管理**：自动根据文件夹生成标签页。
*   **自定义排序**：在管理界面支持表情包文件夹的 **置顶** 和 **上移** 操作，顺序永久保存。
*   **增删改查**：支持添加单张图片到表情包，长按删除单张图片，在管理页删除整个表情包文件夹。
*   **一键分享**：点击表情直接唤起自定义分享菜单，可快速发送至微信/QQ/其他常用App。
*   **数据备份与恢复**：一键导出/导入包含下载链接和包名的 JSON 备份文件，换机/重装无忧。
*   **快捷启动**：通知栏常驻入口，点击即达。

## 🔮 未实现功能 / 待办事项 (欢迎社区接手！)

本项目在开发计划书 V6.0 中，曾设想了一些更为高级的功能，但受限于初学者水平（以及 AI 能力范围），尚未实现。**如果您对本项目感兴趣，非常欢迎 Fork 和贡献代码，共同完善！**

以下是计划书中的部分高阶功能，期待您的实现：

1.  **悬浮窗口 / 悬浮球 (Overlay Window / Floating Widget)** (计划书 ID: INT-304 "启动器替代高风险悬浮窗")
    *   **描述**：实现一个可拖动的悬浮窗或悬浮球，点击后快速打开表情包列表，实现“即开即用，随点随发”的极速斗图体验，替代传统的启动器或通知栏入口。
    *   **挑战**：需要处理系统悬浮窗权限、窗口的生命周期和触摸事件。

2.  **标签页拖拽排序 (UI-203)**
    *   **描述**：在首页的标签页（Tab）上，用户可以按住某个标签进行左右拖拽，以自定义标签页的排列顺序。
    *   **挑战**：Jetpack Compose 的 `TabRow` 不原生支持拖拽，需要实现复杂的手势检测和列表数据更新。

3.  **高级搜索功能**
    *   **描述**：在表情包管理界面，增加搜索框，可根据文件夹名称、图片名称进行模糊搜索。

4.  **云端同步 / 跨设备导入**
    *   **描述**：将备份文件上传至云存储（如 Google Drive, OneDrive）或通过局域网/蓝牙进行跨设备同步。

5.  **GIF 动图优化**
    *   **描述**：目前 GIF 动图显示可能性能不佳，考虑优化加载和播放效率。

## 🛠️ 技术栈

*   **语言**: Kotlin
*   **UI 框架**: Jetpack Compose (Material3)
*   **网络**: OkHttp
*   **图片加载**: Coil
*   **数据存储**: SharedPreferences + Gson
*   **文件操作**: Java Util Zip
*   **核心开发**: niko (人类) & Gemini (AI)

## 🚀 如何安装

1.  下载 `app-debug.apk`。
2.  安装到 Android 手机（需 Android 8.0+）。
3.  首次启动时，请务必授予“通知权限”和“文件读取权限”以获得完整体验。
